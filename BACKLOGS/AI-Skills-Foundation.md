## Architecture Decisions

### Main Principle

KISS: Keep It Simple.

The repository should avoid overengineering and stay practical.

### Skill Structure

Preferred structure for each new skill:

```text
skill-name/
├── SKILL.md
├── templates.md
└── checklist.md
```

### Skill Rules

Each skill should:

- solve one clear task;
- be compact;
- avoid duplicated theory;
- produce practical outputs;
- be usable in real commerce and marketing work;
- support marketplace, website, SEO, content, sales, or growth execution.

## Quality Rules Going Forward

Every new skill should include only:

- `SKILL.md`
- `templates.md`
- `checklist.md`

Every new skill should answer:

1. What task does this solve?
2. When should it be used?
3. What inputs are needed?
4. What workflow should the AI follow?
5. What output should the AI produce?
6. What mistakes should be avoided?
