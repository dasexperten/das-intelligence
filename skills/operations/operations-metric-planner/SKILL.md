# Operations Metric Planner

## Purpose

Define simple operational metrics that show whether work is moving, blocked, late, costly, or improving.

## When to Use

Use this skill when setting dashboards, weekly reports, team reviews, vendor reviews, or process monitoring.

## Inputs

Useful inputs: business area, process, goal, owner, current data, report rhythm, target level, and decision need.

## Workflow

1. Define operational goal.
2. Choose a small set of useful metrics.
3. Map data source and owner.
4. Define review rhythm and decision rule.
5. Recommend dashboard or report format.

## Output

Always provide metric list, definitions, owners, data sources, review rhythm, decision rules, and next actions.

## Common Mistakes

- Too many metrics.
- Metric without action.
- No owner.
- No review rhythm.
