# Automation Runbook Builder

## Purpose

Create runbooks for operating, testing, fixing, and handing off automations.

## When to Use

Use this skill when an automation needs operating instructions, owner notes, test steps, recovery steps, or maintenance rhythm.

## Inputs

Useful inputs: automation name, purpose, tools, schedule, owner, input data, output data, test steps, known issues, and support notes.

## Workflow

1. Define automation purpose and owner.
2. Document tools, schedule, inputs, and outputs.
3. Add test steps and normal checks.
4. Add recovery steps for common issues.
5. Define review rhythm and handoff notes.

## Output

Always provide runbook, owner notes, test steps, recovery steps, review rhythm, and next actions.

## Common Mistakes

- No owner.
- No test steps.
- No recovery notes.
- No review rhythm.
