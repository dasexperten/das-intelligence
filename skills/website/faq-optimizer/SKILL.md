# FAQ Optimizer

## Purpose

Improve FAQ sections so they answer real buyer doubts, reduce support load, support SEO, increase trust, and help visitors move toward purchase or contact.

This skill is built for homepages, product pages, landing pages, checkout, help centers, and international e-commerce websites.

## When to Use

Use this skill when the task is to:

- create or improve FAQ sections;
- answer purchase objections;
- reduce repetitive customer questions;
- support product page conversion;
- improve trust and clarity;
- prepare FAQ content for SEO or schema.

## When NOT to Use

Do not use for legal policy writing or technical schema implementation. Use Schema Markup Specialist for structured data implementation.

## Inputs

Useful inputs:

- page type;
- product or offer;
- customer questions;
- reviews;
- objections;
- support tickets;
- policies;
- delivery and return information;
- SEO keywords;
- target market.

## Workflow

1. Identify the page goal and visitor doubts.
2. Group questions by theme: product, use, trust, delivery, returns, payment, safety, comparison.
3. Remove weak or unnecessary questions.
4. Write clear answers that reduce risk and support conversion.
5. Add proof, links, and next steps where useful.
6. Prioritize FAQ order by buyer decision logic.
7. Recommend schema or internal linking if relevant.

## Output

Always provide:

- FAQ diagnosis;
- recommended question list;
- improved answers;
- order logic;
- missing questions;
- schema notes;
- internal link notes;
- next steps.

## Best Practices

- Answer real objections, not invented filler questions.
- Keep answers short but complete.
- Put purchase-critical questions near the decision point.
- Use FAQ to support trust, not hide important info.
- Link to policies or product details when needed.

## Common Mistakes

- FAQ with generic questions nobody asks.
- Long answers that create more confusion.
- Hiding shipping or return details only in FAQ.
- Repeating page copy without adding clarity.
- Ignoring SEO and schema opportunities.
