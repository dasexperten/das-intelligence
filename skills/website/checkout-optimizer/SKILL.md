# Checkout Optimizer

## Purpose

Improve checkout flows so visitors can complete purchase with less confusion, less risk, fewer unnecessary steps, and higher trust.

This skill is built for e-commerce stores, DTC websites, international checkout flows, carts, payment pages, and delivery-cost steps.

## When to Use

Use this skill when the task is to:

- reduce cart abandonment;
- improve checkout conversion;
- audit cart and checkout friction;
- improve delivery, payment, and return clarity;
- simplify checkout steps;
- diagnose why users stop before payment.

## When NOT to Use

Do not use for payment gateway setup, tax configuration, or legal policy writing. Use this skill for UX, trust, messaging, and conversion improvements.

## Inputs

Useful inputs:

- checkout screenshots or flow;
- cart page copy;
- payment methods;
- shipping rules;
- return policy;
- analytics or abandonment data;
- device type;
- customer objections;
- markets and currencies;
- current checkout steps.

## Workflow

1. Map the checkout steps from cart to confirmation.
2. Identify friction, uncertainty, and forced decisions.
3. Check visibility of shipping cost, delivery time, returns, payment security, and support.
4. Review form length, error messages, and mobile usability.
5. Identify trust gaps near payment.
6. Recommend simplification and copy improvements.
7. Prioritize fixes by impact and implementation effort.

## Output

Always provide:

- checkout diagnosis;
- friction map;
- trust gaps;
- delivery/payment clarity fixes;
- form and UX fixes;
- abandonment reduction ideas;
- priority roadmap;
- KPI notes.

## Best Practices

- Show total cost as early as possible.
- Keep checkout steps clear and predictable.
- Explain delivery, returns, and payment security before payment.
- Reduce unnecessary fields.
- Make error messages specific and helpful.
- Optimize mobile first.

## Common Mistakes

- Surprise shipping cost late in checkout.
- Forcing account creation too early.
- Hiding return policy.
- Weak payment trust signals.
- Too many form fields.
- Poor mobile checkout experience.
