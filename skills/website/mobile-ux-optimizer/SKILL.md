# Mobile UX Optimizer

## Purpose

Improve mobile website experience so visitors can understand, navigate, compare, trust, and convert easily on small screens.

This skill is built for e-commerce, DTC websites, B2B websites, product pages, landing pages, checkout flows, and international mobile traffic.

## When to Use

Use this skill when the task is to:

- improve mobile UX;
- audit mobile page usability;
- reduce mobile bounce;
- improve mobile conversion;
- fix mobile navigation, CTA, readability, or checkout friction;
- adapt desktop pages for mobile behavior.

## When NOT to Use

Do not use for native app design or deep frontend engineering. Use this skill for mobile web UX diagnosis and practical recommendations.

## Inputs

Useful inputs:

- mobile screenshots;
- page URL;
- page type;
- target user;
- traffic source;
- conversion goal;
- device data;
- Clarity recordings;
- GA4 mobile metrics;
- current pain points.

## Workflow

1. Define the mobile user task and conversion goal.
2. Review first-screen clarity, CTA visibility, and navigation.
3. Check readability, tap targets, spacing, and scrolling flow.
4. Review product comparison, forms, cart, and checkout on mobile.
5. Identify hidden trust signals and content overload.
6. Prioritize mobile fixes by impact and effort.
7. Recommend tests and measurement.

## Output

Always provide:

- mobile UX diagnosis;
- first-screen issues;
- navigation issues;
- readability and tap issues;
- mobile conversion blockers;
- priority fixes;
- testing notes;
- KPI notes.

## Best Practices

- Design for thumb navigation and short attention windows.
- Keep primary CTA visible and easy to tap.
- Put trust and delivery information near mobile decision points.
- Reduce large blocks of text.
- Check checkout and forms carefully on mobile.

## Common Mistakes

- Shrinking desktop layout instead of designing for mobile.
- Hiding key CTAs below too much content.
- Small buttons and crowded spacing.
- Long forms.
- Product information that is hard to compare on mobile.
