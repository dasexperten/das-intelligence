# Multilingual SEO Localizer

## Purpose

Plan multilingual SEO so translated pages match local search intent, language, market conventions, hreflang structure, and user expectations.

## When to Use

Use this skill when the task is to localize SEO pages, create multilingual metadata, plan hreflang, adapt keywords by market, or avoid duplicate-language SEO problems.

## Inputs

Useful inputs: source page, target language, target country, keywords, local competitors, page URL structure, hreflang setup, translated copy, and market-specific claims or regulations.

## Workflow

1. Define target country and language.
2. Adapt search intent and query language instead of translating terms literally.
3. Review URL, metadata, headings, content, internal links, and hreflang.
4. Check local proof, units, currency, delivery, claims, and trust signals.
5. Recommend localization fixes and validation steps.

## Output

Always provide localization diagnosis, query adaptation, metadata recommendations, hreflang notes, local trust gaps, internal link notes, and next actions.

## Common Mistakes

- Directly translating keywords.
- Mixing country and language rules.
- Missing hreflang.
- Ignoring local trust and delivery expectations.
- Reusing the same metadata across languages.
